⚡ 5-MINUTE QUICK START
════════════════════════════════════════════════════════════════════════════

1. Push to GitHub (5 min)
   $ git init
   $ git add .
   $ git commit -m "Initial commit"
   $ git remote add origin https://github.com/YOUR_USERNAME/splitwise.git
   $ git push -u origin main

2. Create Neon Database (2 min)
   → Go to neon.tech
   → Sign up with GitHub
   → Create project "splitwise"
   → Copy connection string

3. Deploy on Vercel (3 min)
   → Go to vercel.com
   → Connect GitHub account
   → Select splitwise repo
   → Deploy (takes 2 min)

4. Add Environment Variables (1 min)
   → Vercel Dashboard → Settings → Environment Variables
   → DATABASE_URL = (from Neon)
   → NEXTAUTH_URL = https://your-project.vercel.app
   → NEXTAUTH_SECRET = (generate with: openssl rand -base64 32)

5. Redeploy (1 min)
   → Click "Redeploy" in Vercel
   → Wait 2 minutes
   → Your app is live! 🎉

TOTAL TIME: ~15 minutes (after you read this)

❓ FAQ
════════════════════════════════════════════════════════════════════════════

Q: Will my data be safe?
A: Yes. PostgreSQL is encrypted at rest, Vercel has 99.9% uptime SLA,
   regular backups are automatic.

Q: Can I change the code later?
A: Yes. Just push to GitHub, Vercel auto-deploys.

Q: What if I get 1000 users?
A: Still mostly free. Neon scales automatically ($5-50/month),
   Vercel stays free until extreme scale.

Q: Can friends edit or delete everything?
A: No. Only admins (you) can manage groups. Each user sees their own data.

Q: How do I update the code?
A: Edit locally, git push origin main, Vercel auto-deploys (2 min).

Q: Can I move it to another platform later?
A: Yes. PostgreSQL is standard, code is portable to AWS/Heroku/etc.

Q: Is it secure?
A: Yes. HTTPS by default, passwords hashed with bcrypt,
   JWT tokens for sessions, SQL injection prevented by Prisma.
